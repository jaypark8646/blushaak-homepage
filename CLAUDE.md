# 블루샥 홈페이지 — 작업 규칙

## 자동 커밋 & 배포 링크 안내

이 저장소(`jaypark8646/blushaak-homepage`, master → Vercel 자동 배포)에서 코드를 수정/추가하는 작업을 마치면 **항상** 아래 한 세트로 처리한다.

1. `git add -A` → `git commit -m "..."` → `git push origin master`
2. Vercel 배포 완료까지 폴링 (`vercel ls` / `vercel inspect <url>` 로 "READY" 상태 확인)
3. 배포 완료되면 프로덕션 URL을 **사용자에게 링크로 안내**
   - 예시 문구: `"커밋 푸시하고 배포까지 끝났어요 🍅 https://blushaak-homepage.vercel.app — 여기서 확인해보세요"`

## 중요 원칙

- 사용자가 "커밋", "푸시", "배포", "확인" 을 **명시하지 않아도** 코드 변경 작업 완료 시 자동으로 위 1~3 세트를 실행
- 사용자가 매번 "확인해줘" 라고 요청하지 않아도, 완료 알림에 **배포 링크가 항상 같이 있어야** 함
- 배포가 5분 이상 걸리면 중간 진행 상황 보고 (실패 시 로그 확인 후 재시도하거나 원인 보고)
- 단순 질문/조회/문서 작업(코드 변경 없는) 경우엔 커밋/배포 불필요
